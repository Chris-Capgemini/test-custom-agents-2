## 1. Project Overview

This repository is a **Proof of Concept (PoC) for ALLEGRO modernization** — ALLEGRO being a German social insurance / benefits management desktop application. The PoC demonstrates how a legacy Java Swing desktop client can be integrated with modern web technologies through a WebSocket relay architecture.

The system has **three runtime components** that work together:

| Component | Technology | Purpose |
|-----------|-----------|---------|
| **Node.js Relay Server** | Node.js + WebSocket | Central message bus / broadcast relay on port 1337 |
| **Vue.js Web Client** | Vue 2 + Browser WebSocket | Modern web-based person search and data entry UI |
| **Java Swing Desktop Client** | Java 22 + Tyrus WebSocket | Legacy Allegro desktop form that receives data |

The core use case: a user **searches for a person** in the Vue.js web client, **selects a result and payment data**, clicks "Nach ALLEGRO übernehmen" (Transfer to ALLEGRO), and the selected data **automatically populates the Java Swing Allegro desktop form** via the WebSocket relay.

## 5. Java Swing WebSocket Client — Legacy

**Package:** `websocket`  
**Entry point:** `swing/src/main/java/websocket/Main.java`  
**Lines of Code:** ~457  
**Pattern:** Monolithic (all concerns in one class)

### Overview

The **original PoC implementation**. Everything lives in a single `Main.java` file. This is the first iteration before the MVP refactor. It demonstrates the core WebSocket integration concept but mixes UI construction, WebSocket management, JSON parsing, and business routing into one class.

### Class Structure

```
Main (static)
├── Static UI fields (JFrame, JTextArea, JTextField×10, JRadioButton×3)
├── static JsonParserFactory jsonParserFactory
├── main() → initUI() + connect WebsocketClientEndpoint
├── initUI() → GridBagLayout form construction
├── toSearchResult(String json) → SearchResult
│
├── inner class: WebsocketClientEndpoint [@ClientEndpoint]
│   ├── Session userSession
│   ├── constructor(URI) → ContainerProvider.getWebSocketContainer()
│   ├── @OnOpen onOpen(Session)
│   ├── @OnClose onClose(Session, CloseReason) → latch.countDown()
│   ├── @OnMessage onMessage(String json) → routes to textarea/textfield
│   ├── sendMessage(String message)
│   └── static extract(String json) → Message
│
├── inner record: Message (target, content)
└── inner class: SearchResult (name, first, dob, zip, ort, street, hausnr, ze_iban, ze_bic, ze_valid_from)
```

### Message Routing Logic

The `onMessage` handler uses a `switch` on `message.target`:
- **`"textarea"`** → `textArea.setText(message.content)` — raw text to the text area
- **`"textfield"`** → `toSearchResult(message.content)` → populate all 10 text fields

### JSON Parsing Strategy

Custom streaming JSON parse (no Jackson/Gson) using `javax.json.stream.JsonParser`:
- State machine with boolean flags per field name
- `KEY_NAME` event sets a boolean flag; next `VALUE_STRING` reads the value
- The `extract()` method only handles flat JSON; `toSearchResult()` handles nested person structure

### MVP Data Flow

```
User Action (type in field)
         │
         ▼
   PocView (Swing component)
         │ DocumentListener / ChangeListener
         ▼
   PocPresenter.bind()
         │ model.model.get(prop).setField(value)
         ▼
   PocModel.model (EnumMap)
         │
         │ [User clicks "Anordnen" button]
         │ ActionListener → model.action()
         ▼
   PocModel.action()
         │ Collects all field values → HashMap
         │ httpBinService.post(data)
         ▼
   HttpBinService.post()
         │ HTTP POST → localhost:8080/post
         │ Returns response body
         ▼
   EventEmitter.emit(responseBody)
         │
         ▼
   PocPresenter (subscribed lambda)
         │ view.textArea.setText(responseBody)
         │ Clears all form fields
         ▼
   PocView (UI updated)
```

### Key Design Decisions

| Decision | Implementation | Benefit |
|----------|---------------|---------|
| MVP Pattern | View has no logic; Presenter binds | Testability, separation |
| EnumMap for model state | `EnumMap<ModelProperties, ValueModel<?>>` | Type-safe, complete field coverage |
| EventEmitter for callbacks | Custom observer, not Swing EDT | Decoupled model→presenter notification |
| Generic ValueModel | `ValueModel<String>` and `ValueModel<Boolean>` | Handles mixed field types |
| Java 22 features | `var _` (unnamed variable) | Modern Java idioms |

## 7. Node.js WebSocket Relay Server

**File:** `node-server/src/WebsocketServer.js`  
**Port:** 1337  
**Protocol:** WebSocket RFC 6455  
**Lines of Code:** ~68

### Overview

The simplest possible WebSocket server — a pure **broadcast relay**. Any message received from any client is broadcast to **all** connected clients. This enables the Vue.js web client and Java Swing desktop client to communicate without knowing about each other directly.

### Key Implementation Details

```javascript
// Global state
var clients = [];      // active connections array
var messages = [];     // declared but UNUSED (placeholder for history)

// Connection lifecycle
wsServer.on('request', function(request) {
    var connection = request.accept(null, request.origin);  // accept ALL origins
    var index = clients.push(connection) - 1;               // track position

    connection.on('message', function(message) {
        // Broadcast to ALL clients
        for (var i = 0; i < clients.length; i++) {
            clients[i].sendUTF(json);
        }
    });

    connection.on('close', function(connection) {
        clients.splice(index, 1);    // remove by pre-captured index
    });
});
```

### Important Notes

- **No origin validation** — `request.accept(null, request.origin)` accepts all origins
- **Sender receives own messages** — the broadcast loop includes the sending client
- **UTF-8 only** — binary WebSocket messages are ignored
- **No error handling** — no `wsServer.on('error', ...)` handler registered
- **messages[] is unused** — skeleton for future message history/persistence

## 12. Design Patterns Identified

| Pattern | Where Used | Implementation Quality |
|---------|-----------|----------------------|
| **Model-View-Presenter (MVP)** | `com/poc` Java package | ✅ Clean separation — passive view, presenter as controller |
| **Observer / Event Emitter** | `EventEmitter` + `EventListener` | ✅ Decoupled; supports lambda subscription (Java 22) |
| **Generic Value Wrapper** | `ValueModel<T>` | ✅ Type-safe polymorphic storage in EnumMap |
| **WebSocket Broadcast Relay** | `WebsocketServer.js` | ⚠️ No origin validation, no error handling |
| **In-Memory Mock Repository** | `Search.vue` search_space | ⚠️ Hardcoded — PoC only, no real backend |
| **JSON Envelope Routing** | `onMessage` target switch | ✅ Clean `{target, content}` envelope pattern |
| **Monolithic UI Client** | `websocket/Main.java` | ❌ Legacy pattern — superseded by MVP version |
| **Streaming JSON Parser** | `Main.java` + `HttpBinService` | ⚠️ Verbose manual state machine; no object mapper |
