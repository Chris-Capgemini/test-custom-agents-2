The application is a **three-tier proof-of-concept** that bridges a Vue.js 2.x web search form with a Java Swing desktop application via a Node.js WebSocket relay server. While the MVP architectural intent is sound in the Swing layer, the codebase as a whole carries significant security risks, has zero test coverage, relies on end-of-life frameworks, and contains several bugs that would manifest in production. It should **not be promoted to production** without addressing at minimum the Critical and High items below.

## Section 1 – Node.js WebSocket Server

**File:** `node-server/src/WebsocketServer.js`

### 🔴 CRITICAL — No Origin Validation (ISS-001)

**Current Code (line 32):**
```javascript
var connection = request.accept(null, request.origin);
```

`null` as the first argument accepts **any** sub-protocol, and passing `request.origin` without verifying it means any website can open a WebSocket connection to this server and receive all broadcast data — including IBAN numbers and personal records.

**Fix:**
```javascript

  'http://localhost:8080',
  'http://localhost:8081'   // Vue dev server
];

wsServer.on('request', (request) => {
  if (!ALLOWED_ORIGINS.includes(request.origin)) {
    request.reject(403, 'Forbidden origin');
    console.warn(`Rejected connection from origin: ${request.origin}`);
    return;
  }
  const connection = request.accept(null, request.origin);
  // ...
});
```

### 🔴 CRITICAL — Plain-Text WebSocket (ISS-002)

All data, including IBAN numbers, BIC codes, and personal addresses, travels over an unencrypted `ws://` connection. Any party on the same network can trivially intercept it.

**Fix:** Add TLS termination via a reverse proxy (recommended) or by passing an `https.createServer` instance to the WebSocket library. All client URLs must change from `ws://` to `wss://`.

### 🔴 HIGH — Stale-Index Closure Bug (ISS-004)

**Current Code (lines 35, 62–66):**
```javascript
var index = clients.push(connection) - 1;   // captured at connect time

connection.on('close', function (connection) {
    clients.splice(index, 1);   // WRONG if earlier clients have left
});
```

If clients A, B, C connect (indices 0, 1, 2) and A disconnects first (`clients.splice(0,1)` → array becomes [B,C]), when B later disconnects it still uses its original index `1` — but B is now at index `0`. This silently removes **C** instead of B.

**Fix:**
```javascript
connection.on('close', () => {
  clients = clients.filter(c => c !== connection);
  console.log(`${new Date()} Peer disconnected. ${clients.length} clients remaining.`);
});
```

### 🟠 HIGH — Messages Sent Back to Sender (ISS-005)

**Current Code (lines 55–57):**
```javascript
for (var i = 0; i < clients.length; i++) {
    clients[i].sendUTF(json);   // includes the originating connection
}
```

The Vue textarea watcher fires `sendMessage()` on every keystroke. Each keystroke is relayed back to the same Vue client, which could trigger the watcher again — a feedback loop.

**Fix:**
```javascript
for (const client of clients) {
  if (client !== connection && client.connected) {
    client.sendUTF(json);
  }
}
```

### 🟡 MEDIUM — ES5 Syntax and Outdated Package (ISS-003)

The entire file uses `var` and lacks `'use strict'`. The `websocket` npm package is a pure-JS implementation from 2011; `ws` is the industry standard today.

**Complete modernized server:**
```javascript
'use strict';
import { createServer } from 'http';
import { WebSocketServer } from 'ws';

const PORT = Number(process.env.WS_PORT) || 1337;
const ALLOWED_ORIGINS = (process.env.WS_ALLOWED_ORIGINS || 'http://localhost:8080').split(',');

const httpServer = createServer();
const wss = new WebSocketServer({ server: httpServer });

let clients = new Set();

wss.on('connection', (ws, req) => {
  const origin = req.headers.origin;
  if (!ALLOWED_ORIGINS.includes(origin)) {
    ws.close(1008, 'Forbidden origin');
    return;
  }

  clients.add(ws);
  console.log(`[${new Date().toISOString()}] Client connected from ${origin}. Total: ${clients.size}`);

  ws.on('message', (data) => {
    const message = data.toString('utf8');
    console.log(`[${new Date().toISOString()}] Message received (${message.length} bytes)`);
    for (const client of clients) {
      if (client !== ws && client.readyState === ws.OPEN) {
        client.send(message);
      }
    }
  });

  ws.on('close', () => {
    clients.delete(ws);
    console.log(`[${new Date().toISOString()}] Client disconnected. Total: ${clients.size}`);
  });

  ws.on('error', (err) => console.error(`WebSocket error: ${err.message}`));
});

httpServer.on('error', (err) => { console.error(`HTTP error: ${err.message}`); process.exit(1); });
httpServer.listen(PORT, () => console.log(`WebSocket server listening on port ${PORT}`));
```

### 🔴 CRITICAL — No WebSocket Error/Close Handling (ISS-010, ISS-011)

**Current Code:**
```javascript
connect() {
  this.socket = new WebSocket("ws://localhost:1337/");
  this.socket.onopen = () => { this.status = "connected"; };
  // No onerror, no onclose, no reconnection
}
```

If the WebSocket server is unreachable or drops the connection, the user sees a broken form with no indication of the problem. Additionally, the textarea `watch` fires `sendMessage()` before the socket is confirmed open:

```javascript
watch: {
  internal_content_textarea: function(val) {
    this.sendMessage(val, "textarea");  // may throw if socket not ready
  }
}
```

**Fix:**
```javascript
connect() {
  const WS_URL = process.env.VUE_APP_WS_URL || 'ws://localhost:1337/';
  this.socket = new WebSocket(WS_URL);

  this.socket.onopen = () => {
    this.status = 'connected';
    this.reconnectAttempts = 0;
  };

  this.socket.onerror = (err) => {
    this.status = 'error';
    console.error('WebSocket error', err);
  };

  this.socket.onclose = (evt) => {
    this.status = 'disconnected';
    if (evt.code !== 1000) {
      this.scheduleReconnect();
    }
  };
},

scheduleReconnect() {
  if (this.reconnectAttempts < 5) {
    const delay = Math.min(1000 * 2 ** this.reconnectAttempts, 30000);
    setTimeout(() => { this.reconnectAttempts++; this.connect(); }, delay);
  }
},

safeSend(payload) {
  if (this.socket && this.socket.readyState === WebSocket.OPEN) {
    this.socket.send(JSON.stringify(payload));
  } else {
    console.warn('Cannot send — WebSocket not open');
  }
},
```
